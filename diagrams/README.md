# Diagrams

This folder holds Mermaid source files, rendered exports, thumbnails, and experimental variants for the article **"The First Diagram Is Usually a Liar."**

## Folder Structure

```
/v1/           Round 1 Mermaid source files and contest entries
/v2/           Round 2 — revisions, iterations, and refinements
/exports/      Rendered diagram outputs (PNG, SVG)
/thumbnails/   Scaled-down versions for social and preview use
```

## Iteration Rounds

**V1** corresponds to the first contest or iteration round: diagrams drawn against the initial article thesis with minimal constraint. These reflect the first attempt — which is the whole point.

**V2** corresponds to the second round: diagrams that incorporate feedback, poll outcomes, and council synthesis. These are more refined and more honest.

The comparison between V1 and V2 is part of the argument. V1 is not discarded. It is evidence.

## Source Format

Diagrams are written in [Mermaid](https://mermaid.js.org/) markdown syntax. They render natively in GitHub and can be exported via the Mermaid Live Editor or CLI tools.

Store `.md` or `.mmd` files in `/v1/` and `/v2/`. Store rendered outputs in `/exports/`. Store web/social-ready thumbnails in `/thumbnails/`.

## Naming Convention

Use descriptive, lowercase filenames with hyphens: `flowchart-main-argument-v1.mmd`, `roi-wordcount-comparison-v2.mmd`.
