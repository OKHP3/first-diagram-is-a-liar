## Which Images Were Used Where

Three images were distributed across the LinkedIn publication sequence to
ensure each touchpoint introduced something new:

| Artifact | Images Used |
|---|---|
| Lite post (simultaneous with article) | No images — plain text only |
| Full standalone post | Copilot V1 + Claude V2 |
| Crafted comment | Notion V1 only |
| Article page body | Copilot V1 + Claude V2 |
| Article cover | ETCH-AI-SKETCH cover PNG |

Every reader who went deeper saw a diagram they hadn't seen yet.
No image was wasted. No image appeared in every context.

---

## Lo-res vs Full-res

Lo-res variants (`-lores.png`) are sized for embedding in LinkedIn posts
and comments where file size affects upload reliability. Full-res variants
are the canonical renders suitable for the article page and any print or
high-resolution use.
