## Image Strategy

Three images were distributed across three LinkedIn artifacts to ensure
each touchpoint introduced something new:

- **Lite post** (simultaneous with article publication) — no images; plain text only
- **Full standalone post** — Copilot V1 + Claude V2
- **Crafted comment** — Notion V1 only (not shown elsewhere in the post sequence)

This meant every reader who went deeper saw a diagram they hadn't seen yet.
No image was wasted. No image appeared in all three places.

## Usage

These images are rendered outputs from the Mermaid source files in `../diagrams/`.
The source `.mermaid` files are the canonical versions — these PNGs are provided
for reference and as the exact assets used in the published article and LinkedIn posts.

If you want to render your own version with different theming, use the source files.

## Cover Image Note

The `etch-ai-sketch-cover.png` was originally created as a Sora video.
The static PNG was used instead — the video contained platform watermarks
from a product that has since been discontinued, which would have dated
the content unnecessarily. The PNG conveys the concept cleanly without
the watermark liability.
